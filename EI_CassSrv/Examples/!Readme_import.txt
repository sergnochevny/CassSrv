ИМПОРТ РЕАЛИЗАЦИИ С КАССОВЫХ АППАРАТОВ. (Ver 135)
  пример №1
--------------------------------------------------------------------------------
  Импорт реализации с кассовых аппаратов, в дальнейшем импорт, осуществляется
  через файл определенной структуры.

  Для настройки импорта необходимы следующии элементы:
    1. Database version 135(минимум).
    2. BReadFile.dll(для BTAdmin.exe,BuTuz.exe)
    3. Скрипт UDP_RSTS_CACH_IMPORT_1.
    4. Макет связи "Акт_Реалицация_с_ЭККА.ds".

  Импорт реализован на эвалуаторе. Реализация состоит из юнитов (Units):
    uCashImport      - Главный юнит. Вызывается в событии шапки макета связи
                       OnInsert {cCalcVar('uCashImport','',Chr(59))}.
    uCashImport_Init - Инициализация переменных.
    uCashImport_Buff - Вспомогательный юнит.

  Все юниты представлены в файле "Units.txt". Если на вашей
  some-форме свойство Units не определено, вы можете скопировать содержимое
  файла "Units.txt" и установить юниты.

  ВНИМАНИЕ !!! Инициализация переменных и настройка строк взаимосвязаны.

  Описание переменных:
    uvFileName   - имя файла импорта.
    uvScriptFile - имя файла для скрипта. Файл будет удален после выполнения
                   скрипта.
    uvKlp        - ключ папки.
    uvKld        - ключ документа.
    uvB00Dat     - дата папки в формате 'dd.mm.yyyy'.
    uvBaseCurr   - ключ валюты документа.
    uvAddCurr    - ключ валюты долнительного учета.
    uvC20F3      - RealStr.C20F3.
    uvC20F5      - RealStr.C20F5.
    uvC20F6      - RealStr.C20F6.
    uvPrms       - RealStr.Prms.
    uvUsr        - Код пользователя.
    uvMTU_SD     - ключ МТУ(услуги) для погрешности окрегления суммы
    uvMTU_TsD    - ключ МТУ(услуги) для погрешности окрегления НДС
    uvIncrPW     - ключ подразделения/работника (Пункт погрузки)
    uvIncrPW_R   - 1 или 2 (1-подразделение, 2-работник)
  Большинство переменных выставляются автоматически. Обратите особое внимание на
  следующие переменные:
    uvAddCurr    - если включен дополнительный учет.
    uvC20F3      - переменная должна соответствовать настройке строк
                 (см. TTNSTR_материалы.txt).
    uvC20F5      - просто подумайте.
    uvC20F6      - переменная должна соответствовать настройке строк
                 (см. TTNSTR_материалы.txt).
    uvPrms       - переменная должна соответствовать настройке строк
                 (см. TTNSTR_материалы.txt).
    uvMTU_SD     - карточка должна существовать, NULL - отменяет учет погрешности.
    uvMTU_TsD    - карточка должна существовать, NULL - отменяет учет погрешности.

  ОСОБОЕ ВНИМАНИЕ !!!
    В строковых переменных НЕ используйте символов '@','^','~'. Пользуйтесь
    функцией Chr(<код символа>)
